## B. 核心行为核对

### 1. 菜单栏显示策略

- [x] 有可信周窗口时，菜单栏只显示其百分比（2026-07-18 可视检查：`69%`）
- [ ] 菜单栏不显示 5 小时额度
- [ ] 只有月额度或未知额度时，菜单栏仍显示 `--%`，不会拿其他窗口代替周额度
- [ ] 无真实快照或演示模式时，菜单栏显示 `--%`

### 2. 动态额度展示

- [ ] only-weekly：Popover 只出现周额度，不出现 5 小时固定占位
- [ ] only-monthly：Popover 与 Widget 都显示月额度，且菜单栏保持 `--%`
- [ ] only-unknown：Popover 与 Widget 不生成额度卡片，也不改标为周/月额度
- [ ] 5 小时 + 周 + 月：按该语义顺序显示，数字、进度、状态和恢复时间一致
- [ ] 多个未知窗口：全部隐藏，不占用面板或 Widget 容量
- [ ] 缓存窗口：不生成额度卡片
- [ ] 无效或演示窗口：不生成额度卡片，也不出现伪造的真实百分比或进度
- [ ] 同时返回 canonical 与 fallback 周窗口时只显示 canonical 周额度，不出现“周额度 1 / 2”
- [ ] 超过两行时 Popover 可纵向滚动；打开期间刷新增减窗口后面板尺寸随之更新
- [ ] Widget 小尺寸选择确定性 primary；中尺寸最多展示三个窗口，额外窗口显示 `+N`

### 3. Popover 顶部状态

- [x] 显示更新时间（2026-07-18 可视检查：`18:34`）
- [ ] 当最近尝试时间与成功时间不同，显示 `尝试 ...`
- [ ] 显示数据来源与刷新状态
- [ ] 显示真实链路健康诊断

### 4. 手动刷新

- [x] Popover 中存在“刷新”按钮（2026-07-18 可视/辅助功能检查通过）
- [ ] 刷新期间按钮进入进行中状态且不可重复触发
- [ ] 刷新成功后状态更新
- [ ] 刷新失败后保留上次成功快照

### 5. 真实 / 缓存 / 失败路径

- [x] 真实成功时，Popover 可显示真实数据（2026-07-18 显示周额度 `69%`、最新状态与恢复时间；诊断详情仍需单独人工展开确认）
- [ ] 同一账号和登录会话下，有真实缓存后发生网络/解析失败时，Popover 明确显示“上次成功数据”仍在使用
- [ ] 退出 Codex 登录后，菜单栏、Popover 与 Widget 不再显示退出前额度，菜单栏降级为 `--%`
- [ ] 同账号退出后重新登录，首次新刷新完成前不恢复上一会话额度
- [ ] 切换到另一 ChatGPT 账号时，不合并或展示前一账号的额度窗口、重置时间或重置额度详情
- [ ] `auth.json` 缺失、不可解析或无法提供稳定账号/会话标识时，不把旧缓存作为当前真实数据
- [ ] 无真实快照时，不把占位状态显示成真实额度
- [ ] 真实链路错误至少能区分：
  - [ ] 未找到 codex
  - [ ] 找到 codex 但文件不可执行
  - [ ] Codex app-server 启动失败
  - [ ] Codex 版本 / 必需额度能力不兼容
  - [ ] 需要登录或需要切换到 ChatGPT 账号
  - [ ] 请求超时
  - [ ] 响应不可解析
  - [ ] 通用 RPC / Codex 不可用

### 6. 调度与系统事件

- [ ] 启动后会自动发起刷新
- [ ] 定时刷新仍工作
- [ ] 连续失败后退避仍工作
- [ ] 唤醒后会再次尝试刷新

### 7. 单实例与跨副本仲裁

- [x] 安装版运行时启动带开发绕过的 dist challenger，最终仍只有安装版 owner 进程（2026-07-18 `--verify` 与进程复核通过；唯一菜单栏图标已可视确认）
- [ ] 连续或同时多次 `open -n` 安装版 / 开发版，不出现重复状态栏、并发真实 RPC、持久化覆盖或重复 Widget 更新
- [ ] 后启动副本会显示现有 owner 的 Popover；Popover 已显示时再次启动不会把它关闭
- [ ] owner 正常退出后可由下一次启动接管；异常退出留下的 metadata / 请求不会阻止接管
- [ ] owner 关停期间不再 ACK 新激活请求，且新旧 owner 不会重叠运行业务资源
- [ ] 缺少当前 v2 handoff capability 或内核 `processIdentity` 的旧 schema owner 不会被新版自动终止；新版保持 secondary 并退出，须先正常退出旧 owner 再重新启动新版

### 8. 开机启动与安装路径迁移

- [ ] 只有系统状态为 `enabled` 且本地记录的路径、Bundle ID 与代码签名身份匹配当前 App 时，界面才显示“已启用”
- [ ] `requiresApproval`、`notFound`、身份缺失/损坏或注册仍指向其他副本时，开关不显示有效启用，并给出对应原因
- [ ] 用户明确开启后重复启动 App 或重复打开 Popover，不会再次调用注册；用户明确关闭后不会被自动重新开启
- [ ] 已启用状态下覆盖安装或在同一路径重新签名，下一次启动只执行一次 `unregister → register`，并把身份记录更新到当前 App
- [ ] 将 App 真正移动到新路径（旧路径不存在）后启动新路径，登录项自动校正；下一次登录只从新路径启动
- [x] 首选安装存在时先启动无开发绕过参数的旧副本，旧副本不初始化菜单栏/刷新/Widget，而是启动首选安装并退出（2026-07-18 `--verify` 通过）
- [x] 记录本次旧副本 challenger 的路径、关联 token、首选 owner 路径与 instanceID；重定向日志精确包含同一 token（2026-07-18 `--verify` 通过）
- [x] `--verify` 成功摘要中的最终 PID 仍存活，且精确等于唯一 token 挑战记录的 redirect owner PID；最终 owner record 的 instanceID 也与该 token 对应 instanceID 相同，实际命令路径属于安装版（2026-07-18 通过）
- [x] staging 在旧安装移动前通过版本、主应用/Widget 签名及预期 App Group entitlement 检查；后续失败时恢复 backup，原安装此前运行时尽可能重新启动（成功路径与隔离 rollback 路径均已验证）
- [x] rollback 自身失败时不删除受控工作目录或唯一 backup，并输出可直接执行的人工恢复源路径与目标路径（隔离故障注入通过）
- [x] rollback 前 `pgrep`/`ps` 可判错地确认所有验收进程退出；枚举、PID 身份查询或 TERM/KILL 收敛失败时不移动当前 target、不恢复/重启旧 App，并保留当前安装、backup 与工作目录（隔离故障注入通过）
- [x] `INSTALL_APP_PATH` 只约束 `--verify`；run、debug、logs、telemetry 不因 `/Applications` 中存在 symlink、其他 Bundle ID 或无关损坏目标而被安装路径预检阻断（stub 回归通过）
- [ ] `requiresApproval` 尊重系统设置中的用户选择，不在后台循环注册；用户仍可从 Popover 取消待批准意图

## C. 人工 macOS 行为

### 1. Full-Screen Space

- [ ] 在全屏 Space 中可以打开 Popover
- [ ] 点击外部或失焦后 Popover 会关闭
- [ ] Popover 不会卡成持续悬浮层

### 2. Sleep / Wake

- [ ] 睡眠唤醒后会触发一次刷新尝试
- [ ] 若唤醒后的刷新失败，仍显示上次成功快照
- [ ] 唤醒后的失败类型能在 Popover 顶部读出来

### 3. 空闲资源抽查

- [ ] 空闲时 CPU 接近 `0.0`
- [ ] RSS 维持在原生小工具可接受范围

建议命令：

```bash
ps -axo pid,%cpu,rss,etime,comm | rg 'CodexMonitorNative.app/Contents/MacOS/CodexMonitorNative|CodexMonitorNative$'
```

### 4. 动态窗口视觉与可达性

- [ ] 在 Popover 中检查 1、3、5 个窗口：两列对齐稳定，5 个窗口时最后一行可滚动到达
- [ ] Popover 打开时触发一次窗口集合变化，确认高度更新且没有被裁切的不可达内容
- [ ] 在小/中 Widget 中检查 only-monthly、only-unknown 和重复 known-kind：primary 标签真实，未知窗口不占容量
- [ ] Widget 能量核心只显示动态纯数字；有可信 5 小时时优先显示 5 小时额度，中心不出现 `%`、中文标签或状态
- [ ] Widget 背景细圈、光晕与能量核心严格同心，六个装饰节点围绕核心对称分布

### 5. Popover 键盘与辅助功能

- [ ] 重置额度、字段、诊断和开机启动的行尾控件均为 `24 × 24 pt`、描边粗细一致，并与卡片右缘保持 `10 pt`；默认状态在深色背景上清晰可见
- [ ] 重置额度与开机启动使用同一 `22 pt` 图标列和 `8 pt` 图标—文字间距；刷新与退出文字基线对齐
- [ ] `重置次数`、`最早到期`、`全部 N` 与`开机启动`落在同一文字列；Disclosure 的前导占位不缩小整行点击区
- [ ] 任一披露行整行可点击，悬停与按压反馈清晰；展开/收起时箭头旋转与内容过渡同步，开启“减少动态效果”后立即切换且不丢失状态反馈
- [ ] 在“系统设置 → 键盘”开启“键盘导航”，确保 macOS 会按系统约定把 Tab 焦点交给按钮、开关和披露控件
- [ ] 打开 Popover 时保持内容位于顶部，不因自动聚焦底部操作而发生滚动；首次按 `Tab` 后按阅读顺序进入控件
- [ ] 使用 `Tab` / `Shift-Tab` 可到达当前可见的重置额度披露、开机启动开关、刷新、退出、诊断和字段披露；焦点环清晰可见
- [ ] 聚焦开机启动开关后按空格只切换一次，VoiceOver 将其读作开关类控件并读出“已开启 / 已关闭 / 正在更新”
- [ ] 聚焦任一披露控件后可用系统键盘操作展开或折叠，VoiceOver 同步读出“已展开 / 已折叠”
- [ ] `Command-R` 触发刷新，刷新期间控件不可重复触发且读出“正在刷新”；`Command-Q` 退出应用
- [ ] `Escape` 关闭 Popover，重新打开后不会残留焦点或错误滚动位置
- [ ] VoiceOver 跳过纯装饰能量核心，并能读出顶部状态与更新时间、额度恢复时间及剩余时间
